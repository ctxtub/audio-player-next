# AGENTS

## 目录职责
- 聚合前后端可复用的业务服务层，实现故事与语音等上游服务的封装。
- 提供统一的 HTTP 客户端、错误类型与服务端配置加载逻辑，供 `app/` 路由与 `app/api/` 使用。
- 负责在不同运行时（浏览器、服务端）之间复用协议与校验规则，减少重复实现。

## 子目录结构
- `client/`：浏览器端调用服务端代理的 API 客户端封装，包含故事生成、应用配置与语音合成访问逻辑。
- `http/`：HTTP 基础设施层，抽象 axios 客户端、错误类型与不同运行时的实例化方式。
- `server/`：服务端对接上游 LLM 与 Azure TTS 的适配层，负责环境变量解析、入参校验与异常转换。

## 关键协作与依赖
- 依赖 `@/types/**` 中的请求与响应类型来确保严格的类型约束。
- 与 `app/api/**` server actions 协作，为路由处理器提供故事生成和语音合成能力。
- 通过 `@/lib/http/common` 与上游 RESTful API 通信，统一处理错误并暴露稳定接口。

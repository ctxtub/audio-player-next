# 登录弹窗优化技术方案

## 背景
- 当前登录流程依赖设置页的账号登录弹窗，交互与视觉较为基础，需要依据新设计稿进行体验升级。
- 目标是在保持既有登录功能的前提下，统一卡片化视觉风格，并引入组件库能力以降低维护成本。

## 现状分析
1. **功能结构**：使用自研 `Modal` 结合 antd-mobile 的 `Button`、`Input`、`NoticeBar` 构建账号密码输入与错误提示，按钮竖向排列。
2. **状态管理**：登录态由 `useAuthStore` 维护，弹窗逻辑包含输入清空、禁用态与 loading，流程完整但视觉弱化。
3. **视觉问题**：整体缺乏图标、背景分割与层次强调，间距松散，缺少卡片化观感。

## 设计稿诉求
- 弹窗需呈现浅灰背景的卡片，表单字段使用带左侧图标的圆角输入框，文案为英文占位符（Account / Password）。
- 主按钮为通栏亮蓝色实体样式，整体布局更紧凑；设计稿未展示错误态与取消路径，但实际流程需要覆盖。

## 产品方案
1. **信息架构**
   - 保持顶部标题“账号登录”，在标题下方展示错误提示区域。
   - 表单字段统一为图标输入框组件，包含账号与密码两项。
   - 主 CTA 为通栏蓝色“登录”按钮，补充弱化样式的“取消”次按钮满足现有流程。
2. **交互细节**
   - 登录请求维持 Loading 与禁用反馈；弹窗打开时重置输入内容。
   - 错误提示沿用现有文案逻辑，仅在视觉层面适配新风格。
3. **多端与主题适配**
   - 桌面端保持居中卡片布局，移动端保留边距确保可点击区域。
   - 登录弹窗需同时支持「浅色」「深色」两套主题，确保文案与图标在不同背景下均具备足够对比度。

## UI 优化技术方案
1. **组件选型优先级**
   - 继续使用自研 `Modal` 容器；表单字段优先接入 antd-mobile 的 `Form` + `Input` 组合，若与现有逻辑冲突则保持受控 `Input` 并使用 `List` 或 `Card` 构建卡片。
   - 输入框图标通过 `Input` 的 `prefix` 插槽实现；若图标库不满足则补充项目内 SVG 资源。
2. **结构与布局调整**
   - 在 Modal 内容外包裹卡片容器（优先复用 antd-mobile `Card`），统一内边距与浅灰背景。
   - 按钮区改为主次布局：`Button` 设置 `block` 与 `color="primary"` 用于登录，取消按钮使用 `fill="none"` 或文字按钮样式并置于主按钮下方。
3. **样式适配策略**
   - 更新本地 `index.module.scss` 控制卡片圆角、背景、间距与字体层级，必要时引用现有 CSS 变量以维持主题兼容。
   - 调整 `NoticeBar` 或替换为 antd-mobile `ErrorBlock` / `Form.Error`，通过项目已定义的 `--adm-color-primary`、`--adm-color-danger` 等 Token 贴合设计稿主色与警示色。
   - 主题适配需复用 `styles/index.css` 中的标准变量，例如以 `--card-background`、`--card-border`、`--primary`、`--foreground` 作为浅色/深色卡片底色与文本的基础色值，避免新增登录弹窗专用 Token；如需细化明度，可在局部使用 `color-mix` 结合现有变量调整。
   - 针对图标和占位符统一使用 `--foreground`、`--secondary` 等现有变量的透明度组合，确保在深浅双主题下保持足够对比度。
4. **状态与逻辑复用**
   - 保持 `handleLogin`、`handleCloseModal` 与 `useAuthStore` 现有逻辑，确保功能无回归。
   - 表单受控状态继续沿用现有 hook，减少额外状态分支。
5. **验收与测试**
   - 实施后执行 `yarn lint`、`yarn tsc --noEmit`、`yarn build` 验证质量。

## 风险与注意事项
- 需要确认设计稿未提供的错误态与取消操作是否需要额外视觉稿，如无则根据组件库默认样式适配。
- 若 antd-mobile 组件在 Modal 中出现滚动或焦点问题，应评估是否自定义输入框容器。

# Spec · 播放器页历史区平铺（分段切换面板）

- 日期：2026-06-22
- 状态：已实现
- 配套计划：`docs/plans/2026-06-22-player-history-panel-tiling.md`

## 背景与问题

`/player` 在唱片上方用一张玻璃卡承载「历史记录（提示词历史）」「生成历史」两个按钮，点击各弹出一个 `Modal`。问题：

1. 两个按钮夹在沉浸式唱片上方，与「正在播放」的页面主题不契合。
2. 播放控件下方到底部浮动导航之间有一大片空白（约 180px），页面显得空旷。
3. 历史内容藏在弹窗里，浏览成本高，与「纯播放 + 历史视图」的页面定位脱节。

## 目标

- 移除上方两个历史按钮，把两类历史**平铺**进播放器下方的空白区。
- 用**分段切换**在一块面板内承载两个历史列表，既都保留、又最省空间。
- 历史列表的既有交互（提示词「重新创作」跳创作页、生成历史「回放」、删除、排序、登录门）**行为不变**，仅从弹窗搬到内联。
- 面板精准填满播放器下方剩余视口，空状态也呈齐整占位，消除空旷感。

## 非目标

- 不处理「硬刷新丢播放态」（固有行为，另案）。
- 不改 `PlaybackStatusBoard`（倒计时）、`GenerationPreview`（生成中预览）的逻辑。
- 不动 trackTitle（已于 `f2ecc6b` 修复）。
- 不抽象两个列表的公共样式（超出本次范围，留意但不做）。

## 方案对比

- **A · 分段切换面板（选用）**：一块面板，分段控件在「提示词历史 / 生成历史」间切换，展示当前列表。两者都在、最省空间、改动可控，贴合「把两个按钮做成平铺」的原意。
- B · 单一播放列表：下方只放可回放的「生成历史」。最贴播放器气质，但需把「提示词历史」挪走，且访客态会常驻登录提示。改动范围更大。
- C · 双区堆叠：两个列表都常驻、各占一块。信息最全，但页面变长变满，削弱唱片留白。

选 A：用户明确要「把这两个按钮做成平铺」，A 直接满足且最省版面。

## 设计决策（已自主拍板）

- **默认页**：提示词历史（访客与登录都有内容，首屏不空）。
- **访客 × 生成历史**：内联显示「登录后查看」空状态（与原弹窗一致），不对访客隐藏该 Tab。
- **彻底去弹窗**：本页不再用 `Modal`；不保留「展开大图」入口。`Modal` 组件因 chat `StoryViewer` 仍在用而保留。
- **排序切换上提**：原在弹窗头部的「频率/时间」排序按钮移到面板头部右侧，仅「提示词历史」页显示；排序状态本就在 `promptHistoryStore` 全局，列表组件只读已排序数据。
- **分段控件自研**：项目未引入 antd-mobile Tabs/Segmented，按既有「token 化自定义按钮」风格轻量自研，所有数值引用 Design Token。
- **布局填充（now-playing）**：`.homePage`/`.pageSection` 撑满 `.content` 高度，`HistoryPanel` 以 `flex:1; min-height:0` 吃掉播放器下方剩余空间，列表内部滚动；外层 `.content` 不溢出，避免嵌套滚动；底部沿用 `--tab-bar-safe-bottom` 不被浮动导航遮挡。

## 组件结构

```
app/(main)/player/components/
  HistoryPanel/        # 新增：分段控件 + Tab 状态 + 头部排序槽 + 组合两个内联列表 + onSelectPrompt 跳 /chat
  HistoryRecords/      # 重构：去 Modal/去 showModal ref，导出纯「提示词历史」列表
  GenerationHistory/   # 重构：去 Modal/去 showModal ref，导出纯「生成历史」列表
  InputStatusSection/  # 删除（职责被 HistoryPanel 取代，连带清理整合后残留的死样式）
```

`app/(main)/player/index.tsx`：移除 `InputStatusSection`，在 `AudioPlayer` 下方挂 `HistoryPanel`。

## 验收标准

1. `/player` 唱片上方不再有历史按钮；播放器下方是一块分段切换面板。
2. 分段控件可在「提示词历史 / 生成历史」间切换，默认提示词历史。
3. 提示词历史：点「重新创作」跳 `/chat` 并自动发送、可删除、排序切换在头部生效。
4. 生成历史：登录可见并可「回放」/删除；访客显示「登录后查看」。
5. 面板填满播放器下方空间，空状态为齐整占位，不被浮动导航遮挡。
6. 三道门全绿：`yarn lint` / `yarn tsc --noEmit` / `yarn build`；浏览器活体无报错。

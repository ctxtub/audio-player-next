# AGENTS

## 目录职责
- 承载 Next.js App Router 的页面、API Route 及服务端逻辑。
- 组织按功能拆分的业务模块，向上提供页面和路由入口。
- 通过服务层与 store 交互，协同呈现音频播放业务。

## 子目录结构
- `api/`：实现服务端 API Route，封装应用配置、故事生成与语音合成代理。
- `chat/`：对话页面，提供基于 LLM 的交互式聊天体验。
- `home/`：首页页面实现及本地组件，作为应用主入口。
- `player/`：播放器详情页面，提供沉浸式音频播放体验。
- `setting/`：设置页面，管理应用偏好与用户信息。
- `server/`：Server Actions 与服务端辅助逻辑。
- `services/`：页面复用的业务流程函数。
- `types/`：仅供 `app/` 内部使用的类型定义。
- `utils/`：页面专用的工具函数。
- `layout.tsx` / `page.tsx`：应用根布局及默认页面入口。

## 关键协作与依赖
- 与 `@/components/**`、`@/stores/**` 交互以复用 UI 与状态。
- 借助 `@/lib/**` 完成上游服务调用。
- 与 `@/types/**`、`@/utils/**` 保持类型与工具函数的一致性。
